# 実装計画: 参加者の質問一覧へ集計を統合

## 概要

参加者画面の独立した投票結果カードを削除し、回答済み質問の票数と割合を質問一覧へ表示する。未回答の受付中質問だけ上部に回答フォームを残し、回答済み・終了済み質問は一覧内で結果を確認できるようにする。

## 要件

- 投票後の`VOTE RESULTS`・`FINAL RESULTS`カードを表示しない。
- 回答済み質問の各選択肢へ`1票 (100%)`形式で表示する。
- 未回答質問の集計は公開しない。
- 未回答の受付中質問だけ回答フォームを表示する。
- 終了済みの回答済み質問には最終結果を表示する。

## アーキテクチャ変更

- `src/client/pages/room-page.tsx`: 独立結果表示を削除し、`QuestionOverview`へ質問別集計と投票済みIDを渡す。
- `tests/e2e/room.spec.ts`: 参加者の質問一覧内でリアルタイム結果と最終結果を確認する。

## 実装手順

### フェーズ1: 表示統合

1. **独立結果カードを削除する** (File: `src/client/pages/room-page.tsx`)
   - Action: 回答済みactive質問とclosed質問の`VoteResults`表示を削除する。
   - Why: 質問内容と結果の重複表示をなくすため。
   - Dependencies: なし
   - Risk: 低

2. **質問一覧へ集計を表示する** (File: `src/client/pages/room-page.tsx`)
   - Action: APIから公開された回答済み質問の集計だけ、選択肢の右側へ表示する。
   - Why: 未回答質問の結果を隠しつつ、回答後の結果確認を一覧へ統合するため。
   - Dependencies: ステップ1
   - Risk: 中

## テスト戦略

- E2E: 回答後に質問一覧へ`1票 (100%)`が表示され、終了後も同じ位置に残ることを確認する。
- 静的検査: `pnpm check`
- 回帰テスト: `pnpm test`
- ブラウザテスト: `pnpm test:e2e`
- ビルド: `pnpm build`

## リスクと対策

- **Risk**: 未回答質問の集計が表示される。
  - Mitigation: サーバーが`votedQuestionIds`に一致する結果だけ返す既存制御を維持する。
- **Risk**: 全受付中質問へ回答後、上部が空になる。
  - Mitigation: 投票受付メッセージを表示し、結果の確認先を質問一覧として案内する。
