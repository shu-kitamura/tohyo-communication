# デザインコンセプト

## 1. 全体コンセプト

- **キーワード**: 「盛り上がり」+「リアルタイム」+「一目でわかる」
- **目的**: `03_writing.md` で決めた文言を、開催者・参加者にわかりやすく表示。かつ「盛り上がる」ような形で見せる
- **イメージ**: シンプルで見やすく、リアルタイムの更新が映えるような雰囲気


## 2. カラーパレット

- **プライマリ**: #2563EB（鮮やかなブルー）— 強いコントラストで主ボタンと強調テキストに使用
- **アクセント**: #F97316（オレンジ）— リアルタイム更新のハイライトやアテンションに限定して使用
- **サーフェス**: #F8FAFC（明るいグレー）、#FFFFFF（ベース）、#0F172A（ダークUI用ヘッダー/フッター）
- **テキスト**: #0B1220（メイン）、#475569（サブ）
- **グラフ色**: 4〜6色の固定セット例
  - #2563EB, #10B981, #F59E0B, #EC4899, #8B5CF6, #0EA5E9
- **状態色**: 成功 #10B981 / 警告 #F59E0B / エラー #DC2626 / 情報 #0EA5E9
- 役割を固定し、プライマリとアクセントが隣接しないよう余白か中間色を挟む。

## 3. タイポグラフィ

- **和文/欧文**: Noto Sans JP / Noto Sans（可読性と等幅に近い数字）
- **字重**: 400本文、500ラベル、600ボタン・見出し
- **文字サイズ目安**
  - 見出し（投影用）: 28–32px、行間120%
  - 見出し（モバイル）: 22–24px、行間130%
  - 本文: 16–18px、行間150%
  - ラベル/凡例/ボタン: 14–16px、行間130%
  - 数字（票数/率）: 同サイズで字間-1〜0pxで締める
- **スタイル**: 罫線より余白、下線リンクは避け、強調は太字のみ。全角記号を減らし読みやすさ優先。

## 4. レイアウト・余白

- **グリッド**: 8pxベース。モバイルは左右16pxパディング、投影ビューは左右40–64pxを基本。
- **階層**: H1>H2>本文の三階層。カード/セクション間は24–32px、内部は12–16px。
- **ボタン**: 高さ44–48px、左右16–20px。主ボタンはフル幅（モバイル）/自動幅（デスクトップ）。
- **フォーム**: 入力欄間12–16px、セクション区切り24px。エラーはフィールド直下に表示。
- **グラフ領域**: モバイルは1カラムで幅いっぱい、高さ240–280px。デスクトップは結果+QRを2カラム（約7:3）で配置。
- **ヘッダー/フッター**: 高さ64px目安。投影時はヘッダーを薄くし、グラフを最大化。
- **モーション**: 値更新は0.25–0.35sのスムース、投票到達時にバーを軽くパルス（1.05倍→1.0に戻す）。
