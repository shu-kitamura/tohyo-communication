# ライティング

UI向けの文言ルールと素案。短く案内し、迷わせないことを優先する。

## トーン & 原則

- 敬体で統一。「です・ます」。命令より提案（例: 「送信してください」→「送信します」ボタン）。
- 文は12〜22文字を目安。漢字は詰めすぎず読みやすさを優先。
- 主語は省略し、動詞から始める（例: 「投票を作成」「再接続する」）。
- 状態を即時に伝える単語を優先（送信中/終了/再接続など）。

## ボタン / CTA

- 作成: 「投票を作成」
- 開始/終了: 「開始する」「投票を終了」
- 送信: 「送信する」
- 再接続/再試行: 「再接続」「もう一度試す」
- グラフ切替: 「棒グラフ」「円グラフ」

## 投票作成（主催者）

- タイトル: 「質問を作成」
- プレースホルダー: 質問「例）次のテーマは？」 選択肢「選択肢を入力」
- バリデーション: 未入力「質問を入力してください」 選択肢不足「選択肢を2つ以上追加してください」
- 成功: 「セッションを作成しました」

## 参加者投票

- 見出し: 質問文をそのまま大きく表示。
- 未選択エラー: 「選択肢を選んでください」
- 送信中: 「送信中…」
- 完了: 「投票が完了しました」 ボタン: 「結果を見る」

## 結果表示（主催者）

- 見出し: 「結果を表示中」
- 総投票数: 「総投票数: 24票」など数値先頭で提示。
- グラフ切替ラベル: 「棒グラフ」「円グラフ」
- 終了ボタン: 「投票を終了」 確認モーダルで後述。
- エクスポート: 「ダウンロード」→ボタンを押すと json / csv を選択できる

## 結果表示（参加者）

- 状態文: 「結果を更新中…」「投票は終了しました」
- 凡例/単位: 「票」「%」を明示。凡例は選択肢名のみ。
- 余計な導線は置かず、再投票やホームへのリンクは非表示。

## ステータス / エラー共通

- ネットワークエラー: 「通信が不安定です。再接続してください」
- セッションなし: 「セッションが見つかりません」 サブ: 「URLを確認してください」
- 重複投票: 「この端末では投票済みです」
- 再試行案内: 「もう一度試す」ボタンを隣に置く。

## 通知 / トースト

- 成功: 「保存しました」「送信しました」
- 警告: 「投票は終了しています」
- エラー: 「送信に失敗しました」
- 文字数は18文字以内。アイコンは情報/警告/エラーで色分け。

## モーダル / 確認

- 投票終了: タイトル「投票を終了しますか？」 本文「新しい投票は受け付けなくなります」 ボタン: 「終了する」/「キャンセル」
- エクスポート: タイトル「エクスポートしますか？」 本文「現在の集計をダウンロードします」 ボタン: 「CSVで保存」/「閉じる」

## 用語 / 言い換え

- セッション: 投票ルーム。画面では「投票」と表記して良い。
- 投票終了: 「締め切る」ではなく「終了」で統一。
- 参加者は「あなた」、主催者向けは主語省略で手順を短く。
